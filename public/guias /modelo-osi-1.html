// Conceptos Fundamentales

¿Qué es el Modelo OSI?

El modelo OSI (Open Systems Interconnection) es un marco conceptual que estandariza las funciones de comunicación de un sistema de telecomunicaciones en siete capas abstractas. Desarrollado por ISO en 1984, sigue siendo la referencia universal para entender cómo se comunican los sistemas y dónde actúan los controles de seguridad.

// ¿POR QUÉ ES IMPORTANTE EN SEGURIDAD?
Cada capa del modelo OSI representa una superficie de ataque distinta. Un atacante puede comprometer la comunicación en cualquier capa: desde el cable físico hasta la aplicación del usuario. La defensa en profundidad (Defense in Depth) exige controles de seguridad en cada nivel, no solo en el perímetro.
// MODELO OSI VS TCP/IP
El modelo OSI es teórico y de referencia. En la práctica, Internet usa el modelo TCP/IP de 4 capas (Acceso a red, Internet, Transporte, Aplicación). OSI sigue siendo el estándar de enseñanza porque su granularidad de 7 capas permite identificar con precisión dónde ocurre cada problema o ataque.

El Stack Completo

Encapsulación y Desencapsulación

Al enviar datos, cada capa agrega su propio encabezado (header) al paquete — esto se llama encapsulación. Al recibirlos, cada capa elimina su encabezado — desencapsulación. Este proceso es el camino de cada bit desde la aplicación hasta el cable y viceversa.

// PDU por capa (Protocol Data Unit)
CAPA PDU HEADER AÑADIDO
7 Aplicación Datos Datos de la aplicación (HTML, JSON, XML…)
6 Presentación Datos Cifrado TLS, compresión, conversión de formato
5 Sesión Datos Control de diálogo, sincronización de sesión
4 Transporte Segmento Puerto origen/destino, números de secuencia TCP
3 Red Paquete IP origen/destino, TTL, protocolo
2 Enlace Trama MAC origen/destino, tipo EtherType, CRC
1 Física Bits Señales eléctricas, ópticas o de radio
// RELEVANCIA DE SEGURIDAD
Los atacantes que realizan análisis de tráfico (sniffing) ven la trama completa con todos los headers. Un atacante con acceso a capa 2 puede ver headers de capas 3 y 4. Solo el cifrado de capa 6/7 (TLS) protege el contenido aunque se capture el tráfico. Los firewalls inspeccionan headers de capas 3 y 4; los WAF también inspeccionan capa 7.
// TÉCNICA: PROTOCOL TUNNELING
Los atacantes pueden encapsular tráfico malicioso dentro de protocolos permitidos. Ejemplo: DNS tunneling — exfiltrar datos codificados en consultas DNS (capa 7) para evadir firewalls que solo inspeccionan puertos. Otro ejemplo: ICMP tunneling — enviar datos en el campo de datos de pings ICMP.
// DISPOSITIVOS POR CAPA
Capa 1: Repetidores, hubs, cables
Capa 2: Switches, bridges, puntos de acceso Wi-Fi
Capa 3: Routers, firewalls L3
Capa 4: Firewalls stateful, balanceadores de carga L4
Capa 7: WAF, proxies, balanceadores L7, CDN
// Detalle por Capa — Ataques & Defensas

Las 7 Capas en Detalle

Cada capa tiene su propia función, sus protocolos característicos y sus vectores de ataque específicos. Para cada capa se describen los ataques más relevantes y los controles de defensa correspondientes.

07
Capa de Aplicación
Application Layer · La capa más expuesta
PDU: Mensaje / Datos

Es la capa más cercana al usuario. Proporciona servicios directamente a las aplicaciones: navegación web, correo electrónico, DNS, transferencia de archivos. Al ser la interfaz con el usuario, es la capa más atacada y también donde se implementan los controles más complejos.

// EJEMPLO COTIDIANO Cuando escribes https://banco.com en tu navegador, la capa 7 gestiona la petición HTTP/S, el servidor DNS resuelve el nombre y el servidor web devuelve el HTML. Todo lo que "ves" en pantalla es el resultado de protocolos de capa 7.
HTTP/SDNSFTPSMTPPOP3IMAPSNMPLDAP
⚔️
Ataques
  • SQL Injection — Inyección de código SQL en campos de entrada para manipular o extraer datos de bases de datos.
  • XSS (Cross-Site Scripting) — Inserción de scripts maliciosos en páginas web para robar sesiones o redirigir usuarios.
  • DDoS Capa 7 — Flood de peticiones HTTP aparentemente legítimas para saturar la lógica del servidor.
  • DNS Spoofing — Respuestas DNS falsas que redirigen al usuario a sitios maliciosos.
  • Phishing / BEC — Páginas o emails que imitan servicios legítimos para robar credenciales o dinero.
🛡️
Defensas
  • WAF (Web Application Firewall) — Filtra y bloquea peticiones HTTP maliciosas antes de alcanzar el servidor.
  • Validación de entrada — Sanitizar todos los datos de usuario; usar consultas parametrizadas (prepared statements).
  • DNSSEC — Firma criptográfica de respuestas DNS para prevenir spoofing y cache poisoning.
  • Rate Limiting — Limitar peticiones por IP/usuario para mitigar DDoS y ataques de fuerza bruta.
  • Content Security Policy (CSP) — Cabecera HTTP que restringe fuentes de scripts para bloquear XSS.
06
Capa de Presentación
Presentation Layer · El traductor
PDU: Datos

Actúa como traductor entre la aplicación y la red: convierte datos entre el formato de la aplicación y un formato de red estándar. Se encarga del cifrado y descifrado, la compresión y la conversión de codificaciones (ASCII, Unicode). TLS opera principalmente en esta capa.

// EJEMPLO COTIDIANO Cuando tu banco cifra la conexión con TLS 1.3, es la capa 6 en acción. También cuando el navegador descomprime HTML enviado en formato gzip o convierte una imagen JPEG para mostrarla en pantalla.
TLS 1.3SSL (obsoleto)JPEGPNGMPEGgzipASCII
⚔️
Ataques
  • SSL Stripping — Interceptar y degradar conexiones HTTPS a HTTP sin que el usuario lo note (MitM activo).
  • POODLE / BEAST — Vulnerabilidades en SSLv3/TLS que permiten descifrar datos en tránsito.
  • CRIME / BREACH — Explotar la compresión de datos combinada con cifrado para filtrar tokens secretos.
  • Cipher Downgrade — Forzar el uso de algoritmos de cifrado débiles u obsoletos durante el handshake.
🛡️
Defensas
  • TLS 1.3 obligatorio — Deshabilitar versiones antiguas (SSLv3, TLS 1.0, 1.1) y cipher suites débiles.
  • HSTS (HTTP Strict Transport Security) — Fuerza el uso exclusivo de HTTPS y previene SSL stripping.
  • Certificate Pinning — Anclar el certificado esperado para detectar certificados sustitutos (MitM).
  • Cipher Suites fuertes — Usar solo AES-GCM, ChaCha20-Poly1305 y curvas elípticas modernas (ECDHE).
05
Capa de Sesión
Session Layer · El gestor de conversaciones
PDU: Datos

Gestiona el establecimiento, mantenimiento y terminación de sesiones entre aplicaciones. Controla el diálogo entre sistemas: quién habla cuándo, sincronización y puntos de recuperación. Los tokens de sesión de las aplicaciones web operan en esta capa.

// EJEMPLO COTIDIANO Al hacer una videollamada en Zoom, la capa 5 mantiene la sesión activa, gestiona los turnos de audio/vídeo y permite reanudar si se interrumpe la conexión. El token de sesión que mantiene tu sesión iniciada en el banco también vive aquí.
NetBIOSRPCSIPH.245PPTPSockets
⚔️
Ataques
  • Session Hijacking — Robo del token de sesión para suplantar a un usuario autenticado sin conocer su contraseña.
  • Session Fixation — Forzar al usuario a usar un ID de sesión previamente conocido por el atacante.
  • Replay Attack — Capturar y reutilizar tokens o credenciales de sesión válidas en una nueva solicitud.
  • CSRF (Cross-Site Request Forgery) — Engañar al navegador para que ejecute solicitudes autenticadas no intencionadas.
🛡️
Defensas
  • Tokens de alta entropía — IDs de sesión generados criptográficamente, suficientemente largos y aleatorios.
  • Regenerar sesión post-login — Emitir nuevo token tras autenticación exitosa para prevenir session fixation.
  • Cookies seguras — Flags HttpOnly, Secure y SameSite=Strict en todas las cookies de sesión.
  • MFA (autenticación multifactor) — Reduce el impacto del robo de sesión al requerir segundo factor.
04
Capa de Transporte
Transport Layer · El entregador confiable
PDU: Segmento (TCP) / Datagrama (UDP)

Proporciona transferencia de datos extremo a extremo con control de flujo, control de errores y multiplexación mediante puertos. TCP garantiza entrega ordenada y sin errores; UDP prioriza velocidad sobre fiabilidad. Los números de puerto (80, 443, 22…) son el concepto central de esta capa.

// EJEMPLO COTIDIANO Al descargar un archivo, TCP asegura que cada paquete llega en orden y sin errores. Al jugar online, UDP envía datos de posición sin esperar confirmación para minimizar latencia. El firewall que bloquea el puerto 22 trabaja en capa 4.
TCPUDPSCTPDCCP
⚔️
Ataques
  • SYN Flood — Enviar miles de SYN sin completar el handshake TCP para agotar la tabla de conexiones del servidor.
  • Port Scanning — Escanear puertos abiertos para identificar servicios activos y vulnerabilidades (Nmap).
  • UDP Flood — Saturar puertos UDP con datagramas basura para consumir ancho de banda y CPU.
  • TCP Session Hijacking — Inyectar paquetes con número de secuencia correcto para tomar control de una conexión.
🛡️
Defensas
  • SYN Cookies — Responder al SYN sin reservar estado en la tabla hasta que el handshake se complete.
  • Firewall de puertos — Denegar por defecto todo el tráfico y permitir solo los puertos necesarios (whitelist).
  • Rate limiting de conexiones — Limitar conexiones simultáneas y nuevas por IP origen en el firewall.
  • Scrubbing Centers — Desviar tráfico DDoS masivo a centros de limpieza antes de alcanzar la infraestructura.
03
Capa de Red
Network Layer · El cartero de Internet
PDU: Paquete

Gestiona el enrutamiento lógico de paquetes entre redes distintas. Asigna y gestiona direcciones IP, determina la ruta óptima hacia el destino y fragmenta paquetes según el MTU. Los routers son los dispositivos estrella de esta capa.

// EJEMPLO COTIDIANO Al enviar un email a otro país, los paquetes atraviesan decenas de routers que leen la IP destino y deciden el siguiente salto. Con traceroute puedes ver en tiempo real ese camino de salto en salto.
IPv4IPv6ICMPOSPFBGPIPsecNAT
⚔️
Ataques
  • IP Spoofing — Falsificar la dirección IP origen para ocultar identidad o suplantar hosts de confianza.
  • ICMP Flood (Ping Flood) — Saturar con paquetes ICMP Echo para consumir ancho de banda y CPU.
  • BGP Hijacking — Anunciar rutas BGP falsas para redirigir tráfico de Internet a través de sistemas controlados por el atacante.
  • Fragmentación maliciosa — Enviar paquetes fragmentados irregularmente para evadir firewalls o causar crashes en stacks IP.
🛡️
Defensas
  • Ingress/Egress Filtering (BCP38) — Bloquear en el borde paquetes con IPs origen imposibles, privadas o spoofed.
  • IPsec — Cifrar y autenticar paquetes IP extremo a extremo; previene spoofing e intercepción.
  • RPKI (Route Origin Validation) — Validar criptográficamente los anuncios de rutas BGP para prevenir hijacking.
  • ACLs en routers — Listas de control de acceso para filtrar tráfico por IP origen/destino y protocolo.
02
Capa de Enlace de Datos
Data Link Layer · La red local
PDU: Trama (Frame)

Proporciona comunicación confiable entre dispositivos en la misma red local. Usa direcciones MAC (quemadas en el hardware), gestiona el acceso al medio compartido y detecta errores en tramas. Los switches Ethernet y los puntos de acceso Wi-Fi operan en esta capa.

// EJEMPLO COTIDIANO Cuando tu PC se comunica con el router de casa, usan direcciones MAC. El protocolo ARP resuelve "¿qué MAC tiene la IP 192.168.1.1?" y el switch aprende a qué puerto está conectada cada MAC para no inundar la red con cada trama.
Ethernet 802.3ARPWi-Fi 802.11STPVLAN 802.1QPPP
⚔️
Ataques
  • ARP Poisoning / Spoofing — Enviar respuestas ARP falsas para asociar la MAC del atacante con la IP de la víctima, base del MitM en LAN.
  • MAC Flooding — Saturar la tabla CAM del switch para que opere como hub y exponga todo el tráfico de la red.
  • VLAN Hopping — Saltar entre VLANs mediante doble etiquetado 802.1Q o negociando trunk ports.
  • Evil Twin / Rogue AP — Punto de acceso Wi-Fi falso con el mismo SSID para interceptar tráfico inalámbrico.
🛡️
Defensas
  • Dynamic ARP Inspection (DAI) — El switch valida las respuestas ARP contra la tabla DHCP snooping; descarta ARP falsas.
  • Port Security — Limitar el número de MACs permitidas por puerto de switch; bloquear si se excede.
  • VLAN nativa y segregación — Cambiar VLAN nativa del valor por defecto (1); deshabilitar trunking automático (DTP).
  • 802.1X (NAC) — Autenticar dispositivos antes de permitir acceso a la red local usando RADIUS/EAP.
01
Capa Física
Physical Layer · Los bits en el mundo real
PDU: Bit

La capa más baja: transmite bits como señales eléctricas, ópticas o electromagnéticas sobre el medio físico. Define cables, conectores, frecuencias de radio, voltajes y tasas de transmisión. Sin capa 1, no existe ninguna de las capas superiores.

// EJEMPLO COTIDIANO El cable Cat6 que conecta tu PC al switch, la fibra óptica que llega a tu edificio o la señal 5 GHz del Wi-Fi son capa 1. Un cable dañado o interferencias electromagnéticas causan errores que afectan todas las capas superiores.
Cable UTP/STPFibra ÓpticaWi-Fi RFBluetoothUSBDSL
⚔️
Ataques
  • Wiretapping (escucha física) — Conectar un dispositivo de escucha directamente al cable para capturar tráfico sin modificarlo.
  • Jamming (interferencia RF) — Emitir señales de radio para saturar frecuencias Wi-Fi e impedir comunicaciones inalámbricas.
  • Hardware Implants — Dispositivos físicos insertados en puertos USB o teclados para capturar datos (keyloggers hardware).
  • Corte de infraestructura — Daño deliberado a cables o fibras ópticas para interrumpir comunicaciones (sabotaje).
🛡️
Defensas
  • Seguridad física / CPD — Controlar el acceso físico a salas de servidores, armarios de cableado y puntos de distribución de red.
  • Cables blindados (STP/FTP) — Reducir la posibilidad de escucha electromagnética pasiva en entornos sensibles.
  • CCTV y control de acceso — Monitorear el acceso a infraestructura crítica con registros de auditoría y alarmas.
  • WIDS (Wireless IDS) — Detectar puntos de acceso rogue, jamming y otros ataques en el espectro RF.
// Concepto Clave de Seguridad

Defensa en Profundidad

La defensa en profundidad (Defense in Depth) es la estrategia de implementar controles de seguridad en múltiples capas simultáneamente. Si un atacante supera los controles de una capa, los controles de la siguiente capa lo detienen. El modelo OSI es el marco perfecto para visualizar y planificar esta estrategia.

// Resumen de controles por capa
7 WAF · Validación de entrada · CSP · DNSSEC · Rate limiting
6 TLS 1.3 · HSTS · Certificate Pinning · Cipher suites fuertes
5 Tokens seguros · Cookies HttpOnly/Secure · MFA · Timeout de sesión
4 Firewall stateful · SYN Cookies · Rate limiting L4 · Scrubbing DDoS
3 IPsec · ACLs en routers · BCP38 ingress filtering · RPKI
2 DAI · Port Security · 802.1X · Segregación VLAN · DHCP Snooping
1 Seguridad física · CCTV · Control de acceso CPD · WIDS · STP/FTP
// PRINCIPIO CLAVE
No existe una sola capa que sea suficiente para proteger una comunicación. Un atacante sofisticado intentará múltiples capas hasta encontrar la más débil. La seguridad real requiere controles en todas las capas simultáneamente, no solo en el perímetro.
// RELACIÓN CON ISO 27001
El Anexo A de ISO 27001 mapea directamente con las capas OSI: los controles 8.20–8.22 (seguridad de redes, servicios de red, segregación) cubren capas 2–4. Los controles 8.23–8.26 (filtrado web, criptografía, desarrollo seguro) cubren capas 6–7. Los controles 7.1–7.14 (físicos) cubren capas 1–2.
// MONITOREO POR CAPA
El SIEM recibe eventos de múltiples capas: capa 3/4 desde firewalls y routers (NetFlow, syslog), capa 7 desde WAF, proxies y servidores web. La correlación de eventos de distintas capas permite detectar ataques complejos que evaden controles individuales.
