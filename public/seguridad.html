Seguridad & Alcance

Esta página define claramente qué hace Observo y qué no hace. La idea: reducir “ruido”, evitar malentendidos y entregar confianza con límites explícitos.

✅ Qué hace Observo

  • • Analiza señales públicas/visibles de tu web (TLS/HTTPS, headers, redirecciones, superficie expuesta).
  • • Genera checks verificables y un score orientado a explicación (no a “asustar”).
  • • Guarda una baseline y detecta cambios relevantes entre ejecuciones (diff).
  • • Produce reportes claros para clientes/equipo: qué está bien, qué falta, qué hacer ahora.

❌ Qué NO hace Observo

  • • No realiza pentesting ni explotación de vulnerabilidades.
  • • No fuerza logins, no prueba contraseñas, no hace bruteforce.
  • • No requiere acceso a tu panel, hosting, servidor, DB ni repositorios (modo sin credenciales).
  • • No modifica tu infraestructura ni ejecuta acciones destructivas.

Alcance

Observo está diseñado para “visibilidad + reporting”. Eso significa que su alcance se limita a señales observables desde fuera, y se enfoca en claridad, comparaciones y evidencia.

ENTRADA

Dominio / URL

Observo parte de un dominio. Evalúa lo que un usuario/cliente “podría ver” sin acceso interno.

PROCESO

Checks + baseline

Checks verificables y guardado de baseline para comparar con el tiempo y explicar cambios.

SALIDA

Reporte compartible

Resumen ejecutivo, evidencia simple y recomendaciones priorizadas (quick wins vs estructural).

Score y “No aplica”

Observo puede marcar controles como “No aplica” cuando una capacidad no existe (por ejemplo: no hay login, no hay pagos). La idea es no “castigar” la nota por componentes inexistentes.

Limitación importante

Observo no puede ver lo que está detrás de autenticación, redes privadas o configuraciones internas. Si necesitas evaluación interna, se coordina un assessment/pentest aparte (fuera de Observo).

Datos y privacidad

Qué datos se observan

  • • Metadatos TLS/HTTPS (vigencia, cadena visible, configuración observable).
  • • Headers de seguridad presentes/ausentes y su configuración.
  • • Respuestas HTTP/redirects y señales de superficie expuesta (rutas comunes).
  • • Resultados de comparación (baseline vs ejecución actual).

Qué NO se recopila

  • • No solicitamos credenciales ni tokens.
  • • No exfiltramos información privada ni accedemos a paneles internos.
  • • No ejecutamos payloads maliciosos.
  • • No hacemos ingeniería social.

Transparencia

Si un cliente te pide evidencia, Observo está pensado para mostrar “lo que se verificó” y “por qué importa”, sin humo. Si algo no se puede observar desde fuera, se declara como limitación.

Uso responsable

Observo está diseñado para ayudar a equipos a mejorar y comunicar seguridad, no para abusar de sistemas.

Recomendación práctica

Escanea solo dominios propios o con autorización. Para terceros, usa un acuerdo por escrito (sobre todo si habrá reportes compartidos).
